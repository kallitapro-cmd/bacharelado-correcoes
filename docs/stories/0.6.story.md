# Story 0.6 — ADR-005 — Definir estimativa de custo e hard limit por batch

**Status:** Done
**Sprint:** 0 — ADRs (pré-código)
**Complexidade:** S
**Prioridade:** Máxima
**Dependências:** nenhuma

## Contexto

O sistema usa dois modelos Anthropic com preços distintos:
- **Haiku 4.5**: correção em batch (~$0.25/1M input, ~$1.25/1M output tokens)
- **Sonnet 4.6**: calibração e feedback (~$3/1M input, ~$15/1M output tokens)

Para um PA que processa ~120 alunos por vez, o custo pode variar significativamente dependendo do tamanho dos trabalhos (um PDF de 50 páginas convertido para texto pode ter muito mais tokens do que um PPTX de 10 slides).

O PA precisa ver a estimativa de custo ANTES de confirmar o processamento, para evitar surpresas. Além disso, um hard limit deve ser definido para bloquear batches que excedam um orçamento seguro, protegendo contra uso acidental de volumes altos.

## Objetivo

Criar o ADR-005 que define a fórmula de estimativa de custo por batch, os preços de referência dos modelos, o hard limit padrão e o comportamento quando esse limite é excedido.

## Notas Técnicas

**Modelos e preços de referência (verificar preços atuais em console.anthropic.com):**

| Modelo | Uso no sistema | Input ($/1M tokens) | Output ($/1M tokens) |
|--------|---------------|---------------------|----------------------|
| claude-haiku-4-5 | Correção por lote | ~$0.25 | ~$1.25 |
| claude-sonnet-4-6 | Calibração + feedback | ~$3.00 | ~$15.00 |

**Nota:** Preços Anthropic são cotados em USD. O ADR deve incluir a taxa de câmbio de referência usada e a data, dado que o sistema exibe o limite em R$.

**Estimativa de tokens por aluno (sugestão a validar no ADR):**

| Componente | Tokens estimados |
|------------|-----------------|
| System prompt (SKILL.md) | ~2.000 tokens (constante por batch) |
| Trabalho do aluno convertido para texto | ~1.500 tokens (estimativa média) |
| Response da IA (nota + feedback) | ~500 tokens |

**Fórmula base sugerida:**

```
custo_haiku = N_alunos * (tokens_input * 0.25 + tokens_output * 1.25) / 1_000_000
custo_sonnet = N_alunos * (tokens_input_calibracao * 3.00 + tokens_output_calibracao * 15.00) / 1_000_000
custo_total_usd = custo_haiku + custo_sonnet
custo_total_brl = custo_total_usd * taxa_cambio
```

**Estimativa para 120 alunos (exemplo de cálculo no ADR):**
Usando os valores sugeridos acima + taxa de câmbio ~R$5,50/USD:
- Correção Haiku: 120 × (3.500 × 0.25 + 500 × 1.25) / 1M = ~$0.18
- Calibração Sonnet: ~$0.54 (estimativa)
- Total: ~$0.72 ≈ ~R$4,00

**Hard limit sugerido:**
- Valor padrão: R$20,00 por batch
- Configurável via variável de ambiente `BATCH_COST_LIMIT_BRL`
- Comportamento quando excedido: exibir aviso com estimativa e perguntar se quer continuar (não bloquear automaticamente, pois a estimativa é aproximada)

**Referência cruzada com ADR-003:**
A estimativa de tokens de output depende de `max_tokens` definido no ADR-003 (Story 0.4). Os dois ADRs devem ser consistentes entre si.
