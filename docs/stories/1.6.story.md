# Story 1.6 — Implementar conversor PPTX → texto (python-pptx + fallback OCR)

**Status:** Done
**Sprint:** 1 — Fundação técnica
**Complexidade:** M
**Prioridade:** Alta
**Dependências:** 1.1

## Contexto

Apresentações PPTX são o segundo formato mais comum de entrega no Bacharelado em Administração. Os alunos frequentemente usam Canva ou Google Slides exportados como PPTX — esses arquivos podem ter slides que são 100% imagem (sem text frames), exigindo OCR por slide.

O conversor extrai texto de todos os text frames de todos os slides. Quando um slide não tem text frames (slide-imagem), cada slide é renderizado como imagem e submetido ao OCR com pytesseract.

## Objetivo

Implementar `src/converters/pptx_converter.py` com extração de texto de arquivos PPTX via python-pptx com fallback OCR por slide para apresentações com slides-imagem, mantendo interface de output compatível com os demais conversores.

    """
    Converte um arquivo PPTX em texto.

**Lógica de extração:**
```python
# Pseudocódigo por slide
for slide in presentation.slides:
    texto_slide = extrair_text_frames(slide)  # concatenar shape.text_frame.text
    if len(texto_slide.strip()) < 10:
        # Slide provavelmente é imagem — tentar OCR
        imagem_slide = renderizar_slide_como_imagem(slide)
        texto_slide = pytesseract.image_to_string(imagem_slide, lang='por')
```

**Renderização de slide como imagem:** usar `python-pptx` com `slide.shapes` para detectar slides-imagem. Para renderizar como imagem, pode-se usar `Pillow` para criar uma imagem em branco e desenhar o conteúdo, ou simplesmente detectar `isinstance(shape, Picture)` como indicador de slide-imagem.

[AUTO-DECISION] Para renderização de slides como imagem para OCR, usar detecção heurística: se o slide tem apenas shapes do tipo `Picture` e nenhum `TextFrame` com texto, considerar slide-imagem e acionar OCR. Razão: renderização real via LibreOffice não está disponível no Streamlit Cloud sem configuração adicional, e a heurística cobre o caso de uso do Canva adequadamente.

**metodo_extracao valores:**
- `"nativo"`: todos os slides têm text frames
- `"ocr"`: nenhum slide tem text frames (todos via OCR)
- `"misto"`: alguns slides via text frames, outros via OCR
