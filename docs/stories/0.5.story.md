# Story 0.5 — ADR-004 — Definir política de privacidade e retenção zero de dados de alunos

## Contexto

O sistema manipula dados pessoais de alunos: nome, RA, trabalhos acadêmicos e notas. Todos esses dados são sensíveis sob a LGPD (Lei 13.709/2018) e requerem tratamento explícito quanto ao ciclo de vida, armazenamento e descarte.

A decisão arquitetural adotada é de **retenção zero**: os dados de alunos vivem exclusivamente na sessão Streamlit (`st.session_state`) e são destruídos automaticamente quando a sessão encerra. Nenhum dado é persistido em disco, banco de dados ou serviços externos além da API Anthropic (que processa apenas o conteúdo dos trabalhos, sem PII associada quando possível).

Esta política precisa ser formalizada antes do Sprint 1 para que os desenvolvedores implementem o sistema com os controles corretos desde o início, sem necessidade de retrofit.

## Objetivo

Criar o ADR-004 que formaliza a política de privacidade e retenção zero, define o texto do banner de consentimento, e especifica o que nunca deve aparecer em logs ou mensagens de erro.

## Notas Técnicas

**Dados manipulados pelo sistema e seu ciclo de vida:**

| Dado | Tipo | Onde vive | Por quanto tempo | Descarte |
|------|------|-----------|------------------|---------|
| Nome do aluno | PII | `st.session_state` | Sessão ativa | Automático ao encerrar sessão |
| RA do aluno | PII | `st.session_state` | Sessão ativa | Automático ao encerrar sessão |
| Conteúdo do trabalho | Dado sensível | Memória + API Anthropic (trânsito) | Sessão ativa | Automático ao encerrar sessão |
| Nota gerada | Dado acadêmico | `st.session_state` + Excel em memória | Sessão ativa | Automático ao encerrar sessão |
| Excel gerado | Saída | BytesIO em memória | Até download | Automático após download |
| API key Anthropic | Credencial | Variável de ambiente | Runtime | Nunca persistida |

**Mecanismo de retenção zero:**
- `st.session_state`: destruído automaticamente quando a sessão Streamlit encerra (tab fechada, timeout do servidor, restart)
- `BytesIO`: objeto Python em memória, coletado pelo garbage collector após uso
- Nenhum `open()` para escrita em disco com dados de alunos
- Nenhum `INSERT INTO` ou equivalente com dados de alunos

**Itens que NUNCA devem aparecer em logs (a formalizar no ADR):**
- Número de RA completo (qualquer forma)
- Nome completo do aluno
- Nota numérica associada a um RA específico
- API key (nem prefixo, nem hash)
- Conteúdo do trabalho do aluno

**Política de sanitização de stack traces:**
Quando ocorrer uma exceção durante o processamento de um aluno, o stack trace não deve incluir valores de variáveis que contenham dados do aluno. Usar técnica de sanitização (ex: substituir por `[REDACTED]`) ao capturar a exceção antes de logar.

**Texto sugerido para o banner de consentimento (a validar no ADR):**
```
Este sistema processa dados de alunos (RA, nome, trabalhos e notas) 
exclusivamente para fins de correção acadêmica. 
Os dados são armazenados apenas na sessão atual e são 
automaticamente excluídos ao fechar ou encerrar a sessão. 
Nenhum dado é salvo em disco ou compartilhado com terceiros, 
exceto o conteúdo dos trabalhos enviado à API de IA para correção.
Ao continuar, você confirma que tem autorização institucional 
para processar esses dados.
```

### Pontos fortes

- **Substância acima do mínimo solicitado**: wrapper `safe_logger` com regex patterns concretos prontos para Sprint 1.
- **Defesa em profundidade**: `os.remove()` explícito no `finally` mesmo sabendo que `/tmp` do Streamlit Cloud é zerado entre sessões.
- **Minimização de dados além do exigido**: "evitar enviar RA para Anthropic quando puder ser evitado — usar índice opaco no prompt e desfazer correlação no pós-processamento".
- **Riscos residuais explicitamente aceitos** em tabela (5 entradas com mitigação documentada).
- **Cross-references** com ADR-002 (contrato Clone↔Wrapper) e ADR-005 (orçamento) — coerência arquitetural.
- **Consequências articuladas**: a política de retenção zero simplifica obrigações LGPD (sem DPA, sem notificação ANPD materialmente provável, direito ao esquecimento automático).
