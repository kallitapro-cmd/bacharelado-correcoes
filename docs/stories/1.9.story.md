## Contexto

O ADR-002 (Contrato Clone↔Wrapper) definiu os schemas Pydantic `FichaCorrecao` e `RespostaBatch` como parte do contrato entre o Wrapper Python e o Squad Clone. Esta story implementa esses schemas e os estende com os modelos adicionais necessários para o pipeline completo da aplicação: dados de alunos, arquivos convertidos, estado do batch, etc.

Os schemas são o contrato de dados de todo o sistema. Toda story subsequente que manipule dados depende desta story. A implementação deve ser exatamente fiel ao ADR-002 para os campos `FichaCorrecao` e `RespostaBatch`.

Implementar `src/models/schemas.py` com todos os modelos Pydantic do pipeline, incluindo os schemas definidos no ADR-002 com fidelidade absoluta e os modelos adicionais necessários para o pipeline de dados completo.

## Notas Técnicas

**Schemas do ADR-002 (implementar com fidelidade absoluta):**

```python
# src/models/schemas.py
from typing import List, Literal, Optional
from pydantic import BaseModel, Field, field_validator

class MatrizPontuacao(BaseModel):
    criterio_apresentacao: Optional[float] = Field(None, ge=0.0, le=10.0)
    criterio_conteudo: Optional[float] = Field(None, ge=0.0, le=10.0)
    criterio_metodologia: Optional[float] = Field(None, ge=0.0, le=10.0)
    criterio_conclusao: Optional[float] = Field(None, ge=0.0, le=10.0)

class FichaCorrecao(BaseModel):
    ra: str = Field(..., pattern=r"^\d{11}$", description="RA normalizado 11 dígitos")
    nota_a1: Optional[float] = Field(None, ge=0.0, le=10.0)
    nota_a2: Optional[float] = Field(None, ge=0.0, le=10.0)
    matriz_pontuacao: Optional[MatrizPontuacao] = None
    feedback: str = Field(..., min_length=1)
    flags: List[Literal["plagio", "ia_generativa", "arquivo_errado", "sem_vinculo"]] = Field(default_factory=list)
    confianca: Literal["alta", "media", "baixa"]

    @field_validator("ra")
    @classmethod
    def normaliza_ra(cls, v: str) -> str:
        return v.strip()

class RespostaBatch(BaseModel):
    fichas: List[FichaCorrecao] = Field(default_factory=list)
    observacoes_gerais: Optional[str] = None
```

**Modelos adicionais (não em ADR-002, mas necessários para o pipeline):**

```python
class Aluno(BaseModel):
    ra: str          # RA normalizado (ADR-001)
    nome: str
    email: Optional[str] = None
    telefone: Optional[str] = None

class ArquivoConvertido(BaseModel):
    ra: str
    nome_arquivo: str
    texto: str
    metodo_extracao: str   # "nativo", "ocr", "misto", "nenhum"
    flags: List[str] = Field(default_factory=list)

class ResultadoLote(BaseModel):
    lote_num: int
    fichas: List[FichaCorrecao] = Field(default_factory=list)
    custo_usd: float = 0.0
    status: Literal["pendente", "processando", "concluido", "erro"] = "pendente"

class EstadoBatch(BaseModel):
    disciplina: str
    data_aula: str
    atividade: str
    alunos: List[Aluno] = Field(default_factory=list)
    lotes: List[ResultadoLote] = Field(default_factory=list)
    status_geral: Literal["configurando", "processando", "revisao", "exportado"] = "configurando"
```

**Atenção ao AC sobre RespostaBatch com lista vazia:** o ADR-002 define `fichas: List[FichaCorrecao] = Field(..., min_length=1)`. Esta story deve usar `default_factory=list` sem min_length para permitir lista vazia, pois é necessário para o estado inicial do pipeline. A constraint min_length=1 do ADR-002 se aplica à resposta da IA, não ao estado interno da aplicação.

[AUTO-DECISION] Campo `fichas` em `RespostaBatch` usa `default_factory=list` sem `min_length` no modelo interno de aplicação. O ADR-002 define min_length=1 para o schema de resposta da IA — esse constraint permanece no Wrapper (Story 2.x), não no schema de estado interno. Razão: o pipeline precisa criar um `EstadoBatch` vazio antes de processar qualquer aluno.

## Notas de Implementação

**Decisões autônomas (YOLO):**
- `[AUTO-DECISION]` Uso de `from __future__ import annotations` em `schemas.py` para permitir sintaxe PEP 604 (`str | None`) e PEP 585 (`list[X]`) sob target `py311`. Razão: pyproject.toml define `target-version = "py311"` no ruff.
- `[AUTO-DECISION]` `field_validator("ra", mode="before")` aplicado no RA: strip é executado **antes** da validação por regex `^\d{11}$`. Isso garante que entradas com espaços/quebras de linha sejam normalizadas antes do match, alinhando com ACs 2 e 3.
- `[AUTO-DECISION]` Adicionados aliases `Literal` exportáveis (`FlagCorrecao`, `NivelConfianca`, `StatusLote`, `StatusBatch`) para reuso por outras stories que precisem manipular esses enums sem duplicar literais.
