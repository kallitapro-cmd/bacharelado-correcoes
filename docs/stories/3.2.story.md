# Story 3.2 — Tabela de validação com grupos e plágio (Progressive Disclosure)

**Status:** Done
**Sprint:** 3 — Interface de validação do PA
**Complexidade:** L
**Prioridade:** Alta — tela central da interface do PA
**Dependências:** Story 3.1 (progress_screen — `st.session_state["batch_results"]` disponível); Sprint 2 completo (calibrator, plagiarism_detector, group_detector)

---

## Contexto

Esta é a tela mais complexa do Sprint 3 — onde o PA toma decisões reais sobre as notas dos alunos antes de exportar para a planilha institucional.

O @oalanicolas (2026-05-22) desenhou o fluxo completo da interface de validação usando **Progressive Disclosure** em 3 níveis para evitar sobrecarga cognitiva do PA. A tabela principal exibe apenas o essencial (RA, notas, Δ, status, alertas), com detalhes de plágio e grupos disponíveis sob demanda via expansores.

**Garantia inegociável (ADR-006):** o detector de plágio e o detector de grupos são **informativos** — NÃO modificam notas automaticamente. Toda decisão de nota é do PA. A tabela apenas sinaliza; quem age é o PA.

**Design de fluxo (Alan Nicolas):**
- 84 alunos sem alertas → aprovação em 1 clique (batch approval)
- Alunos com alertas → revisão individual obrigatória antes de liberar exportação
- Export bloqueado até que todas as linhas sinalizadas tenham decisão registrada

---

## Objetivo

Implementar `src/ui/validation_screen.py` como página Streamlit com tabela de validação interativa, Progressive Disclosure de alertas de plágio/grupos, aprovação individual e em lote, edição de notas, e controle de bloqueio de exportação.

---

## Acceptance Criteria

### Tabela Principal

- [ ] **AC-01:** A tabela exibe as seguintes colunas para cada aluno: `RA`, `Nota Bruta`, `Nota Calibrada`, `Δ` (diferença entre nota bruta e calibrada, sinalizada com cor verde/vermelho), `Status` (badge: "Top 10%", "Cap 9" ou "Normal"), `Alertas` (ícones), `Ação`.
- [ ] **AC-02:** A coluna `Alertas` exibe ícones em ordem fixa de gravidade: 🔴 plágio severo (≥ 0.85) → 🟡 plágio moderado (0.70–0.84) → 👥 grupo detectado. Regra de supressão: quando há plágio severo, o ícone 🟡 de plágio moderado do mesmo aluno NÃO é exibido (severo subsume moderado). Quando o aluno tem alerta de grupo E de plágio, ambos os ícones coexistem.
- [ ] **AC-03:** Filtros rápidos via `st.radio` ou `st.selectbox` no topo da tabela: `Todos` | `Com alertas` | `Top 10%` | `Cap 9` | `Normal` — filtram as linhas exibidas sem recarregar o batch.
- [ ] **AC-04:** A tabela é renderizada como lista de linhas via `st.container()` + loop Python (NÃO `st.dataframe` — não suporta botões interativos). Para cada aluno: colunas via `st.columns()`, botão "✓ Aprovar" para alunos sem alertas, botão "👁 Revisar" para alunos com alertas. Ao clicar "Revisar", alterna o flag `st.session_state["revisando"][ra]` (bool). O painel Level 1 é renderizado via `st.container()` — NÃO via `st.expander` nativo, que não preserva estado entre reruns quando contém widgets aninhados.
- [ ] **AC-05:** Existe botão "✓ Aprovar todos sem alertas" (batch approval) visível apenas quando há alunos sem alertas **ainda não decididos**. Ao clicar, exibe confirmação inline antes de executar: `"Confirmar aprovação de X alunos sem alertas? [Confirmar] [Cancelar]"`. Ao confirmar: gera uma entrada por aluno em `st.session_state["decisoes"]` (não entrada agregada) e exibe contador: `"X alunos aprovados em lote"`.
- [ ] **AC-05b:** Após batch approval ou aprovação individual, a coluna "Ação" do aluno aprovado exibe badge `"✓ Aprovado"` (não-clicável) e link `"Desfazer"` que remove a entrada de `st.session_state["decisoes"][ra]` e restaura o botão original — permitindo reversão individual mesmo após aprovação em lote.

### Progressive Disclosure — Nível 1 (razao_confianca)

- [ ] **AC-06:** Para alunos com alertas, ao clicar "👁 Revisar", o painel expandido (Level 1) exibe obrigatoriamente: `razao_confianca` da correção (percentual de confiança do modelo), feedback resumido do squad, e botões de ação do PA ("✓ Manter nota", "✏️ Editar nota", "✗ Marcar revisão manual").
- [ ] **AC-07:** O campo `razao_confianca` é exibido como indicador visual composto: `st.progress(value)` para a barra + `st.markdown` com badge colorido via Streamlit color syntax (`:red[X%]`, `:orange[X%]`, `:green[X%]`) exibindo o percentual numérico ao lado. Faixas: < 60% = vermelho, 60–80% = laranja, > 80% = verde. NÃO usar HTML inline customizado (incompatível com Streamlit Cloud).

### Progressive Disclosure — Nível 2 (pares_suprimidos e grupos)

- [ ] **AC-08:** Dentro do painel de revisão (Level 1), existe link/botão colapsável "Ver detalhes de plágio" (Level 2) que só aparece quando o aluno tem alerta de plágio; ao expandir, mostra tabela de pares com: RA do par, percentual de similaridade, badge de severidade (🔴/🟡).
- [ ] **AC-09:** Dentro do painel de revisão (Level 1), existe link/botão colapsável "Ver detalhes de grupo" (Level 2) que só aparece quando o aluno tem alerta de grupo; ao expandir, mostra: nome do grupo candidato, lista de RAs do grupo, score de confiança do grupo.
- [ ] **AC-10:** Dentro do Level 2 de plágio, os `pares_suprimidos` (similaridade < 0.70) são exibidos como seção secundária cinza: `st.caption("N pares descartados (abaixo do limiar de 70%)")` sempre visível quando `len(pares_suprimidos) > 0`, seguido de toggle `"Ver pares descartados"` que lista os pares em formato tabular reduzido (RA do par, similaridade) com estilo visual atenuado. Quando `pares_suprimidos` está vazio, a seção inteira é omitida. Esta seção NUNCA aparece no Level 0 ou Level 1.

### Edição de Notas e Decisões

- [ ] **AC-11:** Ao clicar "✏️ Editar nota", exibe `st.number_input` com a nota calibrada atual como default (range 0.0–10.0, step=0.1, format="%.1f"); ao confirmar, registra a nota editada em `st.session_state["decisoes"][ra]["nota_final"]` e marca o aluno como revisado. (step=0.1 para granularidade real — step=0.5 limita demais para ajustes finos acadêmicos.)
- [ ] **AC-12:** Toda decisão do PA (aprovar, editar nota, marcar revisão manual) é registrada em `st.session_state["decisoes"]` como dict `{ra: {"acao": str, "nota_final": float, "timestamp": str, "observacao": str}}` — onde `timestamp` segue formato ISO 8601 com timezone Brasil: `datetime.now(ZoneInfo("America/Sao_Paulo")).isoformat()`. Estrutura consumida pela Story 3.3 para gerar o Excel.
- [ ] **AC-13:** Alunos marcados como "Revisão manual" pelo PA são destacados com badge 🔲 "Pendente" e NÃO bloqueiam a exportação — a planilha os exporta com flag `revisao_manual=True`.
- [ ] **AC-13b:** O painel de revisão (Level 1) inclui campo `st.text_area` opcional "Observações" (placeholder: "Anotações para a coordenação..."); o valor preenchido é salvo em `st.session_state["decisoes"][ra]["observacao"]` (string vazia quando não preenchido) — consumido pela Story 3.3 para a coluna `Observações PA` do Excel.

### Controle de Exportação

- [ ] **AC-14:** O botão "Exportar para Excel" fica **desabilitado** enquanto houver alunos com alertas **sem nenhuma das três ações registradas**: Aprovar, Editar nota, ou Marcar revisão manual. "Revisão manual" É uma decisão válida que libera o aluno para exportação (com flag `revisao_manual=True`). Contador exibido: `"X alunos com alertas aguardando decisão (Aprovar / Editar / Revisão manual)"`.
- [ ] **AC-15:** Quando todas as linhas sinalizadas têm decisão, o botão "Exportar para Excel" é habilitado e exibe resumo: `"Y alunos aprovados, Z editados, W marcados para revisão manual"`.

### Sidebar — Zone A (Resumo Agregado)

- [ ] **AC-16:** Sidebar lateral exibe apenas contagens agregadas: total de alunos, alunos aprovados, alunos com alertas pendentes, lotes com erro — SEM detalhar RAs individuais na sidebar (ADR-004: evitar PII em posição proeminente).

### Testes

- [ ] **AC-17:** Existe `tests/unit/test_validation_screen.py` com pelo menos 5 testes cobrindo: (a) construção correta das colunas da tabela a partir de fichas mockadas, (b) filtro "Com alertas" retorna somente alunos com plágio ou grupo detectado, (c) batch approval aprova somente alunos sem alertas e gera uma entrada por aluno em `decisoes`, (d) exportação bloqueada quando há alertas sem decisão, (e) exportação liberada quando todos os alertas têm decisão (incluindo "Revisão manual" como decisão válida).
- [ ] **AC-18:** O filtro ativo (AC-03) é preservado em `st.session_state["filtro_ativo"]` e restaurado a cada rerun — o PA não perde a seleção após aprovar ou editar um aluno. O estado de revisão aberta por aluno (`st.session_state["revisando"]`) também é preservado entre reruns para que o painel de Level 1 não feche ao interagir com widgets dentro dele.

## Notas de Design (Alan Nicolas — 2026-05-22)

**Princípio central:** PA como árbitro final, não como executor de algoritmo.

**Colunas da tabela (ordem visual):**
```
RA | Nota Bruta | Nota Calibrada | Δ | Status | Alertas | Ação
```

**Progressive Disclosure — 3 níveis:**
- **Level 0 (tabela):** RA + notas + Δ + badge status + ícones de alerta — visível para todos
- **Level 1 (expansor ao clicar Revisar):** razao_confianca + feedback resumido + botões de decisão
- **Level 2 (sub-expansor dentro do Level 1):** pares de plágio OU detalhes de grupo — apenas quando relevante

**Regra de supressão de pares_suprimidos:**
- Pares abaixo do threshold NÃO aparecem no Level 2 principal
- Aparecem como "N pares descartados (abaixo do limiar de 70%)" como texto colapsável
- Objetivo: não confundir o PA com falsos alertas já descartados pelo algoritmo

**Batch approval:**
- Visível somente quando há alunos sem alertas pendentes
- Label: "✓ Aprovar todos os X alunos sem alertas"
- Após aprovação em lote: atualiza contador na sidebar em tempo real

**Bloqueio de exportação:**
- Estado: "X com alertas aguardando decisão" (vermelho)
- Estado: "Pronto para exportar" (verde, botão habilitado)
- Transição automática quando contador chega a zero

## Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Performance da tabela com turmas grandes (>100 alunos) | Média | Médio | Usar `st.dataframe` com paginação ou filtros que reduzem linhas visíveis |
| PA editar nota e perder o valor ao rerun | Média | Alto | Armazenar todas as edições em `st.session_state["decisoes"]` imediatamente |
| Alertas de plágio/grupos inconsistentes com fichas calibradas | Baixa | Alto | Cruzar por RA (chave primária) ao montar a tabela, não por índice |
| Progressive Disclosure confusa para PA sem treinamento | Média | Médio | Labels claros: "Ver detalhes de plágio" em vez de "Level 2" |
