# Story 1.12 — Implementar audit log de sessão com download

## Contexto

O ADR-004 (LGPD e Retenção Zero) exige que todas as ações do PA sejam rastreáveis para fins de auditoria, mas que dados pessoais de alunos nunca sejam persistidos entre sessões. O audit log resolve essa tensão: registra quem fez o quê e quando, sem registrar dados de alunos (notas, nomes, RAs).

O log vive em `st.session_state` — memória efêmera da sessão Streamlit. Ao encerrar o browser ou a sessão, o log desaparece (retenção zero). O PA pode baixar o log em CSV antes de encerrar a sessão se precisar de um registro para auditoria institucional.

## Objetivo

Implementar `src/utils/audit_log.py` com registro de ações em memória de sessão, construção de CSV para download, e integração no `app.py` com o botão de download na sidebar.

## Notas Técnicas

**Interface de audit_log.py:**

```python
# src/utils/audit_log.py
import csv
import io
from datetime import datetime, timezone
import streamlit as st

AUDIT_LOG_KEY = 'audit_log'

def _get_log() -> list[dict]:
    """Retorna lista de entradas do audit log da sessão atual."""
    if AUDIT_LOG_KEY not in st.session_state:
        st.session_state[AUDIT_LOG_KEY] = []
    return st.session_state[AUDIT_LOG_KEY]

def log_action(acao: str, payload_resumido: str = "") -> None:
    """
    Registra uma ação do PA no audit log de sessão.

    IMPORTANTE (ADR-004): payload_resumido NÃO deve conter nomes de alunos,
    RAs individuais ou notas. Use resumos agregados como "lote 2, 15 alunos".
    """
    entrada = {
        "timestamp": datetime.now(timezone.utc).isoformat(),
        "identificacao_pa": st.session_state.get("identificacao_pa", "desconhecido"),
        "acao": acao,
        "payload_resumido": payload_resumido,
    }
    _get_log().append(entrada)

def get_csv() -> bytes:
    """Serializa o audit log como CSV UTF-8 para download."""
    log = _get_log()
    if not log:
        return b"timestamp,identificacao_pa,acao,payload_resumido\n"
    output = io.StringIO()
    writer = csv.DictWriter(output, fieldnames=["timestamp", "identificacao_pa", "acao", "payload_resumido"])
    writer.writeheader()
    writer.writerows(log)
    return output.getvalue().encode("utf-8")

def clear_log() -> None:
    """Zera o audit log da sessão."""
    st.session_state[AUDIT_LOG_KEY] = []
```

## Notas de Implementação (Dev)

**Decisões de implementação:**

1. **`datetime.UTC` em vez de `timezone.utc`** — Python 3.11+ expõe o alias `datetime.UTC`, que é a forma idiomática preferida pelo ruff (`UP017`). Comportamento equivalente; apenas migração estética.
2. **`get_csv()` sempre escreve cabeçalho via `DictWriter.writeheader()`** — em vez do branch curto (`if not log: return b"..."`), simplificando o código e evitando drift entre o cabeçalho hardcoded e a lista `CSV_FIELDS`. Para log vazio, o CSV resultante contém apenas a linha de cabeçalho.
3. **`cast` para narrowing do tipo** — `st.session_state` tem tipo `Any` no streamlit; foi adicionado `cast("list[dict[str, str]]", ...)` em `_get_log()` para satisfazer mypy strict sem perder a inicialização lazy.
4. **Constante `CSV_FIELDS`** — definida no módulo para garantir uma única fonte de verdade da ordem dos campos do CSV.
5. **Helper `_log_from()` nos testes** — concentra o `cast` necessário para satisfazer mypy strict em `tests/unit/`, mantendo cada caso de teste enxuto.
6. **Botão de download na sidebar** — colocado após o botão de logout com um `st.divider()` para separação visual. O nome do arquivo inclui `YYYYMMDD-HHMM` para facilitar identificação por sessão.
