# Story 1.7 — Implementar conversor DOCX → texto (python-docx)

**Status:** Done

**Complexidade:** S
**Prioridade:** Alta
**Dependências:** 1.1

## Contexto

Relatórios escritos no formato DOCX são o terceiro formato de entrega mais comum. O formato DOCX (Office Open XML) é bem suportado pelo python-docx, que permite extração direta de parágrafos e tabelas sem necessidade de OCR.

O formato `.doc` legado (binário Microsoft Word pré-2007) não é suportado pelo python-docx e deve retornar um erro claro ao invés de falhar silenciosamente. Esta distinção é importante pois alunos mais antigos podem tentar entregar `.doc`.

## Objetivo

Implementar `src/converters/docx_converter.py` com extração de texto de arquivos DOCX via python-docx, incluindo parágrafos e tabelas, com tratamento explícito do formato `.doc` legado.

## Notas Técnicas

**Assinatura da função principal:**
```python
def convert_docx(file_path: str | Path) -> dict:
    """
    Converte um arquivo DOCX em texto.

    Returns:
        dict com campos:
        - texto (str): texto extraído concatenado de parágrafos e tabelas
        - paragrafos (int): número de parágrafos extraídos
        - metodo_extracao (str): "nativo"
        - flags (list[str]): ["arquivo_ilegivel"], ["formato_nao_suportado"] ou []
    """
```

**Detecção de .doc legado:**
```python
# Por extensão
if str(file_path).lower().endswith('.doc') and not str(file_path).lower().endswith('.docx'):
    return {"texto": "", "paragrafos": 0, "metodo_extracao": "nenhum",
            "flags": ["formato_nao_suportado"]}
```

**Extração de tabelas:**
```python
for table in doc.tables:
    for row in table.rows:
        for cell in row.cells:
            texto += cell.text + " "
```

**Ordem de extração:** parágrafos primeiro, depois tabelas (concatenar com separador `\n`).
