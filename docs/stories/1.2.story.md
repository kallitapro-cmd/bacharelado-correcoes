# Story 1.2 — Implementar autenticação com streamlit-authenticator + audit trail

**Status:** Done

**Complexidade:** M
**Prioridade:** Alta
**Dependências:** 1.1

## Contexto

A aplicação do Corretor Acadêmico lida com notas e feedbacks de alunos. Qualquer acesso não autenticado expõe dados acadêmicos e cria risco LGPD. A autenticação protege toda a aplicação e garante que apenas Professores Assistentes (PAs) autorizados acessem o sistema.

O ADR-004 (LGPD e Retenção Zero) exige que antes do login o usuário veja um banner de consentimento explícito e que toda interação seja associada a uma identificação do PA para fins de auditoria. O campo de identificação não é uma credencial de segurança — é apenas um nome/identificador para o audit trail.

O mecanismo de lockout (5 tentativas → 15min de bloqueio) não é fornecido nativamente pelo streamlit-authenticator e deve ser implementado via `st.session_state`.

## Objetivo

Implementar autenticação completa com `streamlit-authenticator`, incluindo banner LGPD, campo de identificação do PA, lockout por tentativas excessivas e integração no `app.py` como guarda global de todas as telas.

## Notas Técnicas

**Texto do banner LGPD (ADR-004 — wording exato):**
> "Este sistema processa notas e feedbacks de atividades acadêmicas. Os dados são utilizados exclusivamente para correção automatizada e não são armazenados em servidores externos. Sua sessão não persiste dados pessoais após o encerramento. Ao continuar, você confirma que tem autorização para acessar os dados das turmas."

**Estrutura de config.yaml.example:**
```yaml
credentials:
  usernames:
    pa_exemplo:
      email: pa@exemplo.edu.br
      name: Professor Assistente Exemplo
      password: hashed_password_aqui  # usar Hasher do streamlit-authenticator
cookie:
  expiry_days: 1
  key: chave_secreta_aleatoria_aqui
  name: corretor_academico_cookie
preauthorized:
  emails: []
```

**Mecanismo de lockout (session_state):**
- `st.session_state['login_attempts']` — contador de tentativas
- `st.session_state['lockout_until']` — timestamp Unix do fim do bloqueio
- Verificar antes de exibir o formulário; zerar após login bem-sucedido

**Referência:** ADR-004 (`docs/decisions/ADR-004-lgpd-retention.md`) — regras de privacidade e texto do banner.
