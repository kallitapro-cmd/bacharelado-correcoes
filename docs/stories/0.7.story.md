## Contexto

O sistema inclui detecção de similaridade entre trabalhos do mesmo batch usando `difflib` (biblioteca padrão Python). Essa funcionalidade levanta flags para o PA quando dois ou mais trabalhos são muito similares, indicando possível plágio ou cópia.

A política de detecção precisa ser formalizada antes do Sprint 1 porque:
1. O threshold de similaridade é uma decisão pedagógica/institucional, não técnica pura.
2. O sistema **nunca deve** punir automaticamente — apenas alertar o PA.
3. O comportamento para trabalhos em grupo é ambíguo sem uma política explícita.
4. A detecção de IA generativa é uma questão em aberto que precisa de decisão explícita (implementar ou não, e com qual abordagem).

## Objetivo

Criar o ADR-006 que define a política completa de detecção de plágio: threshold, comportamento dos badges, tratamento de grupos e decisão explícita sobre detecção de IA generativa.

## Notas Técnicas

**Ferramenta de detecção:**
`difflib.SequenceMatcher` da biblioteca padrão Python. Calcula a razão de similaridade entre dois textos (0.0 a 1.0). Vantagens: sem dependências externas, rápido para batches de 120 alunos, funciona offline.

**Threshold sugerido (a validar no ADR):**
≥70% de similaridade → flag de plágio interno. Justificativa: trabalhos com mais de 70% de conteúdo idêntico raramente são coincidência em contexto acadêmico, mas o threshold pode variar por tipo de atividade (questões objetivas vs. dissertativas).

**Comportamento de comparação:**
- Comparação é feita entre todos os pares dentro do mesmo batch (complexidade O(n²))
- Para 120 alunos: ~7.140 comparações — aceitável em tempo de processamento
- O resultado é uma matriz de similaridade; pares acima do threshold são flaggados

**O que o PA vê (badge de plágio):**
- Badge vermelho no card do aluno
- Ao clicar: lista dos pares similares com porcentagem de similaridade
- Decisão de punição 100% do PA: o sistema não altera a nota

**Política para grupos (questão em aberto):**
Se a atividade é em grupo, trabalhos do mesmo grupo DEVEM ser similares. Opções:
1. O sistema não sabe quais alunos são do mesmo grupo → flaga todos os pares acima do threshold (falso positivo para grupos)
2. A planilha do PA inclui coluna "grupo" → sistema ignora comparações dentro do mesmo grupo
3. O PA configura grupos manualmente antes do processamento

O ADR deve decidir entre essas opções ou propor uma quarta alternativa.

**Detecção de IA generativa (questão em aberto):**

Opções para o ADR:
- **Implementar heurística simples**: detectar padrões comuns de texto gerado por IA (frases muito uniformes, ausência de erros, vocabulário padronizado) — baixa precisão, muitos falsos positivos
- **Usar API externa**: GPTZero, Originality.ai, etc. — custo adicional, dependência externa, dados saem do sistema
- **Deferir para Sprint 2**: não implementar no MVP, apenas documentar a flag no schema (campo `flags` no ADR-002) para uso futuro
- **Não implementar**: documentar decisão de não detectar IA no MVP

O ADR deve escolher uma dessas opções com justificativa.

**Garantia explícita de não-punição automática:**
O ADR deve incluir uma seção "Garantias do Sistema" afirmando que:
- Nenhuma nota é reduzida automaticamente por suspeita de plágio
- Nenhum aluno é reprovado automaticamente por suspeita de IA generativa
- Toda ação é exclusivamente do PA após análise do badge
