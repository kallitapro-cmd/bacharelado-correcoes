# Story 2.1 — Implementar `clone_client.py` — Wrapper Python do squad

**Status:** Ready
**Sprint:** 2 — Pipeline de batch de correção
**Complexidade:** M
**Prioridade:** Alta
**Dependências:** 2.0 (testes unitários de `clone_client.py` — Status: Ready)

---

## Contexto

O ADR-002 (`docs/decisions/ADR-002-clone-contract.md`) define o contrato entre o Wrapper Python e o squad `@corretor-academico`. A revisão de Sprint 2 (Pedro Valério, 2026-05-22) identificou 9 veto conditions (V1-V9) que precisam ser implementadas no módulo antes de entrar em produção. A Story 2.0 já registrou os 5 testes unitários que cobrem os caminhos críticos das funções `build_system_prompt()` e `_detect_injection()` — esses testes devem existir e passar antes de marcar esta story como concluída.

O ADR-003 (`docs/decisions/ADR-003-temperature-policy.md`) define `temperature=0` para a fase de correção em batch, garantindo determinismo nas notas (exigência inegociável para auditorias acadêmicas).

Esta story implementa o módulo `packages/wrapper/clone_client.py` com o pseudocódigo completo do ADR-002 revisado, juntamente com os arquivos de suporte (`exceptions.py`, `schemas.py`, `__init__.py`).

---

## Objetivo

Implementar `packages/wrapper/clone_client.py` e os módulos de suporte (`exceptions.py`, `schemas.py`, `__init__.py`) conforme o contrato do ADR-002 revisado (Sprint 2), cobrindo todas as 9 veto conditions (V1-V9). O módulo deve passar em todos os 5 testes da Story 2.0 ao ser integrado.

---

## Story

**As a** desenvolvedor(a) do pipeline de correção acadêmica,
**I want** um Wrapper Python que leia os arquivos do squad como system prompt e chame a API Anthropic com todas as veto conditions do ADR-002 tratadas,
**so that** o orquestrador de batch possa corrigir turmas inteiras com determinismo, resiliência e custo controlado.

## Acceptance Criteria

- [ ] **AC-01:** `SQUAD_ROOT` usa path absoluto resolvido relativo ao `__file__` (V1) — NUNCA relativo ao CWD
- [ ] **AC-02:** `_read_squad_file()` levanta `FileNotFoundError` se arquivo ausente; levanta `EmptySquadFileError` se conteúdo vazio após `.strip()` (V4); usa fallback latin-1 com log de warning se `UnicodeDecodeError` em UTF-8 (V7)
- [ ] **AC-03:** `build_system_prompt()` concatena os 8 arquivos do squad na ordem correta com delimitador `## SOURCE: {path}` (exatamente conforme pseudocódigo ADR-002, seção "Implementação do Wrapper")
- [ ] **AC-04:** `_detect_injection()` testa os 3 padrões de `INJECTION_PATTERNS` do ADR-002 e retorna bool
- [ ] **AC-05:** `corrigir_aluno()` verifica `stop_reason == "max_tokens"` ANTES de tentar validar JSON (V3) — levanta `ClonTruncatedResponseError`
- [ ] **AC-06:** Retry de validação inclui `raw[:500]` + lista de erros Pydantic no prompt de retry (V2) — máximo 1 retry de validação (2 tentativas totais)
- [ ] **AC-07:** `API_TIMEOUT = 90` passado explicitamente como parâmetro `timeout` na chamada `client.messages.create` (V6)
- [ ] **AC-08:** System prompt enviado com `cache_control: {"type": "ephemeral"}` dentro do array `system` (V9)
- [ ] **AC-09:** `BATCH_SIZE = 10` definido como constante com comentário explicando V8 (equilíbrio custo de retry × overhead de chamadas)
- [ ] **AC-10:** `temperature=0.0` conforme ADR-003 (fase de correção = determinístico)
- [ ] **AC-11:** `except ValidationError` narrowado (não bare `except Exception`) — MNT-001 do Sprint 1 implementado aqui
- [ ] **AC-12:** Todos os 5 testes da Story 2.0 passam com `pytest tests/unit/test_clone_client.py`

### Padrões do projeto (Sprint 1 / Sprint 2)

- Encoding: UTF-8 por padrão; fallback latin-1 apenas com `logger.warning` (ADR-002 V7)
- Imports: absolutos (`from packages.wrapper.schemas import ...`)
- Logs: usar `logging.getLogger(__name__)` — nunca `print()`
- Sem hardcode de paths relativos ao CWD (ADR-002 V1)
- `except ValidationError` (pydantic) — nunca bare `except Exception` para esse caso (MNT-001 Sprint 1)

### Nota sobre sqlite3 e paths efêmeros (Sprint 2 pattern)

Esta story não usa sqlite3, mas o padrão obrigatório do Sprint 2 se aplica a qualquer path de arquivo temporário: usar `/tmp/` em vez de paths relativos ao projeto.

### Referências cruzadas com testes da Story 2.0

Os testes em `tests/unit/test_clone_client.py` cobrem:
- TC-01: `build_system_prompt()` com todos os 8 arquivos → verifica `result.count("## SOURCE:") == 8`
- TC-02: `build_system_prompt()` com arquivo ausente → `FileNotFoundError` mencionando `blocklist-bajulacao.md`
- TC-03: `build_system_prompt()` com arquivo vazio → `EmptySquadFileError` mencionando `blocklist-bajulacao.md`
- TC-04: `_detect_injection()` com padrão de injection → `True`
- TC-05: `_detect_injection()` com texto acadêmico normal → `False`

Os testes usam `monkeypatch` para substituir `SYSTEM_PROMPT_SOURCES` por paths temporários (`tmp_path`). A implementação deve expor `SYSTEM_PROMPT_SOURCES` como variável de módulo para que `monkeypatch.setattr` funcione.
