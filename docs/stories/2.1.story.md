# Story 2.1 — Implementar `clone_client.py` — Wrapper Python do squad

**Status:** Done
**Sprint:** 2 — Pipeline de batch de correção
**Complexidade:** M
**Prioridade:** Alta
**Dependências:** 2.0 (testes unitários de `clone_client.py` — Status: Ready)

---

## Contexto

O ADR-002 (`docs/decisions/ADR-002-clone-contract.md`) define o contrato entre o Wrapper Python e o squad `@corretor-academico`. A revisão de Sprint 2 (Pedro Valério, 2026-05-22) identificou 9 veto conditions (V1-V9) que precisam ser implementadas no módulo antes de entrar em produção. A Story 2.0 já registrou os 5 testes unitários que cobrem os caminhos críticos das funções `build_system_prompt()` e `_detect_injection()` — esses testes devem existir e passar antes de marcar esta story como concluída.

O ADR-003 (`docs/decisions/ADR-003-temperature-policy.md`) define `temperature=0` para a fase de correção em batch, garantindo determinismo nas notas (exigência inegociável para auditorias acadêmicas).

Esta story implementa o módulo `packages/wrapper/clone_client.py` com o pseudocódigo completo do ADR-002 revisado, juntamente com os arquivos de suporte (`exceptions.py`, `schemas.py`, `__init__.py`).

---

## Objetivo

Implementar `packages/wrapper/clone_client.py` e os módulos de suporte (`exceptions.py`, `schemas.py`, `__init__.py`) conforme o contrato do ADR-002 revisado (Sprint 2), cobrindo todas as 9 veto conditions (V1-V9). O módulo deve passar em todos os 5 testes da Story 2.0 ao ser integrado.

## Story

**As a** desenvolvedor(a) do pipeline de correção acadêmica,
**I want** um Wrapper Python que leia os arquivos do squad como system prompt e chame a API Anthropic com todas as veto conditions do ADR-002 tratadas,
**so that** o orquestrador de batch possa corrigir turmas inteiras com determinismo, resiliência e custo controlado.
