# Story 3.3 — Exportação para Excel com decisões do PA

**Status:** Done
**Sprint:** 3 — Interface de validação do PA
**Complexidade:** S
**Prioridade:** Média — última etapa do fluxo; desbloqueada somente após Story 3.2
**Dependências:** Story 3.2 (validation_screen — `st.session_state["decisoes"]` e `st.session_state["batch_results"]` disponíveis)

---

## Contexto

Após o PA revisar e aprovar as notas na tabela de validação (Story 3.2), ele precisa exportar os resultados para a planilha institucional `.xlsx` que será entregue à coordenação. Esta story fecha o fluxo completo do Sprint 3.

O Excel gerado deve ser compatível com o formato da planilha institucional (Aba 1 = dados de alunos, Aba 2 = notas e feedback). Alunos marcados como "revisão manual" são exportados com flag explícita para que a coordenação saiba quais registros precisam de verificação humana.

O design prioriza **portabilidade**: o arquivo gerado deve ser abrível no Excel, LibreOffice e Google Sheets sem perda de formatação básica.

---

## Objetivo

Implementar `src/ui/export_screen.py` como página Streamlit que consolida as decisões do PA (`st.session_state["decisoes"]`), as fichas calibradas e os alertas, gera um arquivo `.xlsx` via `openpyxl`, e disponibiliza para download via `st.download_button`.

---

## Acceptance Criteria

- [ ] **AC-01:** Ao entrar na tela, verifica `st.session_state["decisoes"]` e `st.session_state["batch_results"]`; se ausentes, exibe `st.error` e link "← Voltar à validação" sem gerar o arquivo.
- [ ] **AC-02:** O arquivo Excel gerado contém duas abas: **"Notas"** (dados principais) e **"Alertas"** (detalhes de plágio e grupos para referência do PA).
- [ ] **AC-03:** Aba "Notas" contém as colunas: `RA`, `Nome` (reunido de `st.session_state["nomes_por_ra"]` — mapa criado na Story 3.0), `Turma`, `Nota Bruta`, `Nota Calibrada`, `Nota Final` (nota decidida pelo PA via Story 3.2), `Status` (Top 10%/Cap 9/Normal), `Revisão Manual` (True/False), `Observações PA` (campo `observacao` do dict `decisoes[ra]`, string vazia quando ausente).
- [ ] **AC-04:** Aba "Alertas" contém as colunas: `RA`, `Tipo de Alerta` (Plágio/Grupo/Ambos), `Detalhes` (pares de plágio como texto concatenado, ou nome do grupo), `Decisão do PA` (ação tomada: Aprovado/Editado/Revisão Manual).
- [ ] **AC-05:** Alunos com `revisao_manual=True` têm a linha na aba "Notas" destacada com preenchimento amarelo (`openpyxl.styles.PatternFill`) para facilitar identificação visual na coordenação.
- [ ] **AC-06:** O arquivo é disponibilizado via `st.download_button` com nome padrão `notas_{turma}_{data}.xlsx` onde `turma` vem do metadata da Tela 1 e `data` é a data atual no formato `YYYYMMDD`; se `turma` estiver vazia, usa `"turma"` como fallback.
- [ ] **AC-07:** Antes de gerar o arquivo, a tela exibe um resumo de conferência: total de alunos, aprovados, editados, revisão manual, lotes com erro — permitindo que o PA confirme antes do download.
- [ ] **AC-08:** A geração do Excel NÃO envolve nenhuma chamada à API externa — é processamento 100% local (ADR-004: dados dos alunos não saem do sistema após o batch).
- [ ] **AC-09:** Existe `tests/unit/test_export_screen.py` com pelo menos 3 testes: (a) geração do Excel com fichas mockadas produz arquivo com 2 abas, (b) aluno com revisao_manual=True aparece na aba "Notas" com flag correta, (c) nome do arquivo gerado segue padrão `notas_{turma}_{data}.xlsx`.

## Notas de Design (Alan Nicolas — 2026-05-22)

**Fluxo Tela 4 — Exportação:**
- Zone A (topo): resumo de conferência (contagens por categoria)
- Zone B (centro): preview inline das primeiras 10 linhas da aba "Notas"
- Zone C (rodapé): botão de download + opção "← Corrigir mais alunos" (reset da sessão)

**Princípio de fechamento de loop:**
- PA vê o resumo ANTES de baixar — não há surpresas
- Botão de download é a última ação, não a primeira
- "← Corrigir mais alunos" limpa `st.session_state` e volta para Tela 1

**Separação de responsabilidades:**
- `excel_builder.py` contém APENAS lógica pura de construção do workbook (sem Streamlit)
- `export_screen.py` contém APENAS a UI (sem lógica de negócio de Excel)
- Essa separação permite testar a geração do Excel sem contexto Streamlit

## Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| `openpyxl` não instalado no ambiente | Baixa | Alto | Verificar `requirements.txt` inclui `openpyxl>=3.1` |
| Nome de turma com caracteres especiais quebra o nome do arquivo | Média | Baixo | Sanitizar turma com regex `[a-zA-Z0-9_-]` antes de usar no nome |
| Arquivo Excel corrompido por dados inesperados (None, NaN) | Média | Alto | `excel_builder.py` converte todos os valores para str/float antes de escrever |
