# Story 1.4 — Validar conectividade e modelos da API Anthropic

**Status:** Done
**Sprint:** 1 — Fundação técnica
**Complexidade:** S
**Prioridade:** Alta
**Dependências:** 1.1

## Contexto

O Corretor Acadêmico depende da API Anthropic para toda a lógica de correção. Antes de construir o pipeline de batch (Sprint 2+), é crítico garantir que o SDK está corretamente instalado, a conectividade funciona, e os dois modelos definidos no ADR-003 (`claude-haiku-4-5-20251001` para batch e `claude-sonnet-4-6` para calibração/revisão) respondem corretamente.

Esta story cria um wrapper mínimo (`api_client.py`) que será reutilizado pelas stories de Sprint 2+ para todas as chamadas à API. O wrapper isola a configuração da API key e expõe uma interface simples para testes de conectividade.

## Objetivo

Implementar `src/utils/api_client.py` com função `test_connection()` que verifica se a API Anthropic está acessível e se ambos os modelos definidos no ADR-003 respondem. Documentar em testes que a conectividade foi validada.

## Acceptance Criteria

- [x] `src/utils/api_client.py` implementado com função `test_connection() -> bool` que retorna `True` se a API responde
- [x] `test_connection()` realiza chamada real ao modelo `claude-haiku-4-5-20251001` com prompt mínimo (ex: "Diga 'ok' em uma palavra") e verifica que a resposta é não-vazia
- [x] Chamada real bem-sucedida para `claude-sonnet-4-6` documentada nos testes (`tests/unit/test_api_client.py`)

**Modelos definidos no ADR-003:**
- `claude-haiku-4-5-20251001` — modelo padrão para batch (custo/performance)

```python

MODEL_REVIEW = "claude-sonnet-4-6"

def get_client() -> Anthropic:
    """Retorna cliente configurado com API key dos secrets do Streamlit."""
    ...

def test_connection(model: str = MODEL_BATCH) -> bool:
    """Testa conectividade com a API Anthropic. Retorna True se OK."""
    ...
```

**Tratamento de erros (conforme ADR-002 — política de retry):**
- `anthropic.AuthenticationError` → mensagem clara: "API key inválida ou ausente. Verifique ANTHROPIC_API_KEY nos secrets."
- `anthropic.APIConnectionError` → mensagem clara: "Sem conectividade com a API Anthropic. Verifique sua conexão."
- Nunca logar a API key ou qualquer parte dela (ADR-004 — safe_logger, Story 1.11).

**Parâmetros de temperatura para testes (ADR-003):**
- Usar `temperature=0` para chamadas de teste (determinismo)
- `max_tokens=4096` global (ADR-003)

**Referências:**
- ADR-002 (`docs/decisions/ADR-002-clone-contract.md`) — contrato de chamada, retry policy, error handling
- ADR-003 (`docs/decisions/ADR-003-temperature-policy.md`) — modelos, temperatura, max_tokens
