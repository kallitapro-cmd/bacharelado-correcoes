# Story 2.2 — Implementar `batch_processor.py` — Orquestrador de lotes

**Status:** Ready
**Sprint:** 2 — Pipeline de batch de correção
**Complexidade:** M
**Prioridade:** Alta
**Dependências:** Story 2.1 (clone_client.py implementado)

---

## Contexto

O ADR-002 define o padrão Wrapper Python como única forma de integração com o squad de correção. A Story 2.1 implementa `clone_client.py`, responsável por uma única chamada à API Anthropic para um lote de alunos. Esta story implementa a camada acima: o orquestrador `batch_processor.py`, que recebe a lista completa de alunos vinda da Aba 1 da UI, particiona em lotes de `BATCH_SIZE=10` (ADR-002, veto V8), distribui o trabalho via `ThreadPoolExecutor`, persiste estado intermediário em sqlite3 efêmero e retorna os resultados consolidados para a UI.

O ADR-005 define a política de orçamento: estimativa calculada **antes** de qualquer chamada à API, com bloqueio automático (`OrcamentoExcedidoError`) se o custo estimado ultrapassar `MAX_COST_BRL`. Isso garante que nenhuma chamada de produção ocorra sem aprovação prévia do PA.

O Sprint 1 registrou o item de débito técnico MNT-003: strings livres sendo passadas como `acao` para `log_action()`. Esta story resolve esse débito introduzindo o enum `AcaoBatch` como contrato formal para todos os payloads de log do batch.

---

## Objetivo

Implementar `src/batch/batch_processor.py` — orquestrador que recebe a lista de alunos (da Aba 1), particiona em lotes de `BATCH_SIZE=10`, chama `clone_client.corrigir_aluno()` via `ThreadPoolExecutor`, persiste estado intermediário em sqlite3 efêmero e retorna resultados para a UI. A estimativa de custo é calculada previamente (ADR-005) e o processamento é bloqueado se o custo estimado exceder `MAX_COST_BRL`.

## Acceptance Criteria

- [ ] **AC-01:** `processar_batch(alunos, enunciado, metadata)` particiona a lista em lotes de `BATCH_SIZE=10` (do ADR-002) — sem hard-code do valor 10 diretamente no código; usa a constante `BATCH_SIZE` importada ou definida no módulo
- [ ] **AC-02:** Processamento de lotes usa `ThreadPoolExecutor` com `max_workers` configurável (default 3)
- [ ] **AC-03:** Estado intermediário de cada lote é persistido em sqlite3 efêmero em `/tmp/batch_state.db` com o comentário EXATAMENTE conforme o padrão obrigatório do SPRINT-2-INDEX.md:
  ```python
  # EFÊMERO: dados perdidos ao reiniciar sessão — comportamento esperado (ADR-004)
  DB_PATH = Path("/tmp/batch_state.db")  # NUNCA usar caminho relativo ao projeto
  ```
- [ ] **AC-04:** Falha de um lote (`ClonValidationError` ou `ClonTruncatedResponseError`) NÃO aborta os demais lotes — a exceção é capturada, registrada no banco e o processamento continua nos lotes restantes
- [ ] **AC-05:** `ClonTruncatedResponseError` marca os alunos afetados como `"status": "REVISÃO MANUAL — TRUNCADA"` no banco sqlite3
- [ ] **AC-06:** `ClonValidationError` marca os alunos afetados como `"status": "REVISÃO MANUAL — VALIDAÇÃO"` no banco sqlite3
- [ ] **AC-07:** Estimativa de custo é calculada ANTES de qualquer chamada à API, usando a fórmula do ADR-005 (`N_lotes × custo_haiku + 1 × custo_sonnet + N_regeneracoes × custo_regen`), e retornada junto com os resultados finais
- [ ] **AC-08:** Se `custo_estimado_brl > MAX_COST_BRL`, o processamento é BLOQUEADO antes de qualquer chamada à API — levanta `OrcamentoExcedidoError` com mensagem clara conforme ADR-005 (inclui: estimativa calculada, hard limit, excesso, instruções para ajuste)
- [ ] **AC-09:** `MAX_COST_BRL` lido de `st.secrets.get("MAX_COST_BRL", "15.0")` com fallback para `15.0` (padrão ADR-005; compatível com Story 1.3 — leitura de secrets via Streamlit)
- [ ] **AC-10:** MNT-003 do Sprint 1 resolvido: `acao` passada ao `log_action()` usa constantes do enum `AcaoBatch` (não strings livres) — implementar `AcaoBatch(str, Enum)` em `src/batch/batch_processor.py`
- [ ] **AC-11:** `log_action()` chamado em: início do batch, por lote processado (com numero do lote e total), e ao final do batch — payloads sem PII (ex: `"lote 3/12, 10 alunos"`, não nomes nem RAs)
- [ ] **AC-12:** `pytest tests/unit/test_batch_processor.py` passa com mocks de `clone_client.corrigir_aluno`

## Padrão sqlite3 OBRIGATÓRIO

O seguinte trecho DEVE aparecer no código exatamente com este comentário e este caminho:

```python
# EFÊMERO: dados perdidos ao reiniciar sessão — comportamento esperado (ADR-004)
DB_PATH = Path("/tmp/batch_state.db")  # NUNCA usar caminho relativo ao projeto
```

**Motivação:** o Streamlit Cloud free tier usa filesystem efêmero destruído a cada redeploy. sqlite3 aqui serve exclusivamente como estado intermediário de sessão de batch — não como persistência cross-session. Uso de `/tmp/` é intencional e documentado.

**Anti-padrão proibido:**
```python
# ERRADO — sobrevive ao redeploy apenas localmente; perde dados em produção silenciosamente
DB_PATH = Path("batch_state.db")
```

## Notas de Implementação

### Estrutura da função principal

```python
def processar_batch(
    alunos: list[dict],
    enunciado: str,
    metadata: dict,
    max_workers: int = 3,
    n_regeneracoes: int = 3,
) -> dict:
    """
    Orquestra a correção em lotes.

    Returns:
        {
            "resultados": [...],      # lista de resultados por aluno
            "custo_estimado_brl": float,
            "lotes_processados": int,
            "lotes_com_erro": int,
        }

    Raises:
        OrcamentoExcedidoError: se custo_estimado_brl > MAX_COST_BRL
    """
```

### Enum AcaoBatch (resolve MNT-003)

```python
from enum import Enum

class AcaoBatch(str, Enum):
    BATCH_INICIADO = "batch_iniciado"
    LOTE_PROCESSADO = "lote_processado"
    LOTE_ERRO = "lote_erro"
    BATCH_CONCLUIDO = "batch_concluido"
```

**Uso correto (MNT-003 resolvido):**
```python
log_action(acao=AcaoBatch.BATCH_INICIADO, payload={"total_alunos": len(alunos), "n_lotes": n_lotes})
```

**Anti-padrão proibido (era MNT-003):**
```python
log_action(acao="batch_iniciado", ...)  # ERRADO — string livre
```

### Fórmula de estimativa (ADR-005)

```python
PRECO_HAIKU_IN  = 0.80   # USD por 1M tokens de entrada
PRECO_HAIKU_OUT = 4.00   # USD por 1M tokens de saída
PRECO_SONNET_IN = 3.00   # USD por 1M tokens de entrada
PRECO_SONNET_OUT = 15.00 # USD por 1M tokens de saída
TAXA_CAMBIO     = 5.50   # USD → BRL (referência maio/2026 — ADR-005)
TOKENS_IN_HAIKU = 4_000
TOKENS_OUT_HAIKU = 4_096
TOKENS_IN_SONNET = 3_000
TOKENS_OUT_SONNET = 4_096

def estimar_custo_brl(n_lotes: int, n_regeneracoes: int = 3) -> float:
    custo_haiku  = n_lotes        * (TOKENS_IN_HAIKU  * PRECO_HAIKU_IN  + TOKENS_OUT_HAIKU  * PRECO_HAIKU_OUT)  / 1_000_000
    custo_sonnet = 1              * (TOKENS_IN_SONNET * PRECO_SONNET_IN + TOKENS_OUT_SONNET * PRECO_SONNET_OUT) / 1_000_000
    custo_regen  = n_regeneracoes * (TOKENS_IN_HAIKU  * PRECO_HAIKU_IN  + TOKENS_OUT_HAIKU  * PRECO_HAIKU_OUT)  / 1_000_000
    total_usd = custo_haiku + custo_sonnet + custo_regen
    return total_usd * TAXA_CAMBIO
```

### Leitura de MAX_COST_BRL

```python
import streamlit as st

def _get_max_cost_brl() -> float:
    try:
        return float(st.secrets.get("MAX_COST_BRL", "15.0"))
    except (ValueError, TypeError):
        # valor inválido → usa padrão e loga aviso (ADR-005)
        return 15.0
```

### Schema sqlite3 sugerido

```sql
CREATE TABLE IF NOT EXISTS batch_lotes (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    lote_num    INTEGER NOT NULL,
    total_lotes INTEGER NOT NULL,
    status      TEXT    NOT NULL,  -- 'OK', 'REVISÃO MANUAL — TRUNCADA', 'REVISÃO MANUAL — VALIDAÇÃO'
    alunos_json TEXT,              -- JSON dos resultados do lote (sem PII desnecessária)
    criado_em   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Isolamento de falhas por lote (AC-04)

Cada future do `ThreadPoolExecutor` deve ser envolvido em try/except dentro do loop de coleta de resultados. `ClonValidationError` e `ClonTruncatedResponseError` devem ser capturados individualmente — outros erros inesperados podem ser re-levantados ou registrados dependendo da política de logging.

## Riscos

- **Risco 1:** Story 2.1 (`clone_client.py`) pode não estar Done quando esta story for iniciada. Mitigação: esta story depende explicitamente de 2.1 — o `@dev` deve verificar a existência e interface de `clone_client.corrigir_aluno()` antes de implementar. Se 2.1 estiver em InProgress, pode-se criar um stub mínimo e prosseguir com mocks nos testes.
- **Risco 2:** `st.secrets` não disponível fora do contexto Streamlit (ex: ao rodar testes unitários). Mitigação: `_get_max_cost_brl()` usa try/except e os testes devem monkeypatchar essa função para injetar valores de teste diretamente.
- **Risco 3:** Condições de corrida no sqlite3 com `ThreadPoolExecutor`. Mitigação: usar `threading.Lock` para serializar escritas no banco ou usar WAL mode (`PRAGMA journal_mode=WAL`).
- **Risco 4:** Preços Anthropic podem mudar — constantes de estimativa ficarão desatualizadas. Mitigação: documentadas com data de referência (ADR-005); ADR deve ser revisitado a cada 6 meses.
